# ValueShield — H股价值投资助手 v1.0

> **设计哲学：算法为辅，主观为主；数据本地化；移动端优先。**
>
> 用户画像：A/H 股价值投资者，遵循格雷厄姆"烟蒂股"策略，管理 20+ 只标的。
> 核心痛点：港股通无法自动网格交易，需通过软件实时监控，辅助用户在手机端（同花顺/银河）完成手动"分批买入、分批卖出"的闭环。

---

## 一、已实现功能（v1.0 现状）

### 1. 智能监控与时间管理
- 仅在 **周一至周五 09:15–12:00、13:00–16:10** 轮询行情
- 自动识别中国法定节假日，进入静默模式（基于 `chinese_calendar` 库）
- 默认每 30 秒轮询一次（`config.json` 可调）
- 动态股息率（TTM）= 近12个月分红额 / 现价，实时展示

### 2. 灵活网格算法（Grid Engine）
- 初始公式：`Step = (Base_Price − Hist_Min) / 20`，共 20 格
- 支持在 Web 界面随时修改 `Base_Price` 或手动指定 `Step`，自动重算全部触发价
- 每笔买入独立追踪成本，默认止盈 7%，支持针对单笔持仓自定义止盈比例

### 3. "影子网格"同步系统（手动闭环）
- Bark 推送含 Web 回调链接，用户手机操作完成后点击链接跳转
- Web 看板点击【确认成交】，`state.json` 立即原子写入
- 支持【手动补录】格子，处理程序未捕捉到的手动操作

### 4. 压力测试预警
- 实时计算：剩余风险资金 = Σ(未占用格子触发价 × 每手股数)
- 超过 `cash_reserve` 设定值时，Web 首页红色横幅预警 + Bark 推送

### 5. Web 看板（Streamlit 深色金融风格）
- **热力网格图**：20 格可视化，已持仓高亮（绿色），当前价位（蓝色指针）
- **持仓列表**：每笔显示买入价、实时盈亏%、止盈价、一键确认卖出
- **参数面板**：Base/Step 重置、全局设置（Bark Token、现金预留、轮询间隔）
- 移动端适配，手机浏览器可用

## 二、技术架构

```
ValueShield/
├── app.py          # Streamlit Web 看板（主入口）
├── monitor.py      # 后台监控轮询循环
├── engine.py       # 网格算法引擎（GridEngine、Holding）
├── crawler.py      # AkShare 行情/分红数据获取（含3次自动重试）
├── notifier.py     # Bark API 推送（买入/止盈/风险预警）
├── config.json     # 静态配置（标的参数、Bark Token 等）
├── state.json      # 实时持仓与网格状态（自动原子写入）
├── requirements.txt
└── tests/
    ├── ut/         # 单元测试（109个，覆盖 engine/crawler/notifier/monitor）
    └── sct/        # 场景测试（29个，端到端完整交易流程）
```

**技术栈：** Python 3.12 · AkShare · Streamlit · Requests · chinese_calendar

**数据安全：** `state.json` 采用 `.tmp` + `os.replace()` 原子写入，防止掉电丢失状态。

## 四、快速启动

```bash
# 安装依赖
pip install -r requirements.txt

# 方式A：仅 Web 看板（手动刷新）
streamlit run app.py --server.port 8501

# 方式B：后台监控 + Web 看板（完整模式）
nohup python3.12 monitor.py > monitor.log 2>&1 &
streamlit run app.py --server.port 8501 --server.address 0.0.0.0
```

**首次配置：** 编辑 `config.json`，填入 `bark_token` 和 `web_server_url`（局域网 IP）。

## 五、待讨论 / 潜在改进方向

以下是目前版本的局限性和可以讨论的改进点：

### 🔧 功能层面
- [ ] **多账户支持**：目前单账户，能否区分不同券商账户的持仓？
- [ ] **A股支持**：当前仅针对港股（HKD 计价），是否需要扩展 A 股？
- [ ] **历史成交记录**：卖出后的记录目前只在内存，能否持久化到 SQLite 并展示收益曲线？
- [ ] **自动同步持仓**：能否对接同花顺/富途 API 自动读取实际持仓，减少手动补录？
- [ ] **多格子同时触发保护**：股价跳空大幅下跌时，一次触发多格，是否需要"单次最多确认 N 格"的限制？
- [ ] **分红再投资**：分红到账后是否自动调整 `annual_dividend_hkd` 参数？

### 📊 算法层面
- [ ] **非均匀网格**：目前是等距格子，是否考虑价格越低格距越小（加密底部）？
- [ ] **动态 Base 自动跟踪**：股价长期上涨后，Base 是否应自动上移（趋势追踪）？
- [ ] **止盈策略多样化**：除了固定比例止盈，是否支持"跌破某价格止盈"（移动止盈）？
- [ ] **ATR 止损**：是否需要基于波动率的止损线？

### 📱 UI/UX 层面
- [ ] **交易日历**：Web 看板展示本周/本月已完成交易和盈亏汇总
- [ ] **推送频率控制**：同一格子触发后冷却多久才再次推送？
- [ ] **Telegram 推送**：作为 Bark 的备用方案（Bark 仅支持 iOS）

### ⚙️ 部署层面
- [ ] **Docker 化**：提供 Dockerfile，一键部署到云服务器
- [ ] **HTTPS**：通过 nginx 反向代理 + Let's Encrypt 实现外网安全访问
